Chapter 4: Optimizing Model Performance - Hyperparameter Tuning

Chapter 4: Hyperparameter Tuning

While model parameters (like the θ coefficients) are learned from the data during training, there is another class of parameters that are not. These are called hyperparameters—the "dials" and "knobs" that you, the practitioner, set before the training process begins to control the model's behavior.

What are Hyperparameters?

Choosing the right hyperparameters is a critical step that often marks the difference between a high-performing, generalizable model and a useless one. Common examples include:

  • The regularization strength alpha in Ridge or Lasso Regression.
  • The inverse regularization strength C in Logistic Regression.
  • The number of neighbors (n_neighbors) in a K-Nearest Neighbors model.
  • The maximum depth of a tree (max_depth) in Decision Trees and Random Forests.
  • The learning rate in a neural network.
Finding the optimal set of hyperparameters is a process known as hyperparameter tuning or hyperparameter optimization.

Strategies for Hyperparameter Tuning

Grid Search

An exhaustive search that systematically tries every possible combination of hyperparameter values you define in a "grid".

Pro: Guarantees finding the best combination within the grid.
Con: Suffers from the "curse of dimensionality"; becomes incredibly slow as you add more hyperparameters to tune.

Random Search

Instead of trying all combinations, it samples a fixed number of combinations randomly from the specified hyperparameter space.

Pro: Often finds a better model in less time than grid search because it doesn't waste time on unimportant parameters.
Con: No guarantee of finding the absolute best combination.

Bayesian Optimization

A "smarter" approach. It builds a probabilistic model of the relationship between hyperparameter values and model performance. It then uses this model to intelligently select the next set of hyperparameters to try, focusing on regions most likely to yield improvements.

The Critical Role of Cross-Validation

A critical problem arises during tuning: if we use our test set to evaluate each hyperparameter combination, we will inadvertently "leak" information from the test set into our model selection process. The hyperparameters we choose will be optimized for that *specific* test set, and our final performance estimate will be overly optimistic.

The Golden Rule: The test set must remain untouched until the very end, used only once for the final evaluation of the chosen model.

The solution is K-Fold Cross-Validation. It provides a robust procedure for model evaluation and tuning by splitting the *training set* into smaller subsets.

Implementation in Scikit-Learn

Scikit-learn provides powerful tools that automate the process of combining a search strategy with cross-validation.

GridSearchCV

Performs an exhaustive grid search with cross-validation.

from sklearn.model_selection import GridSearchCV
from sklearn.linear_model import LogisticRegression

# 1. Define the model
model = LogisticRegression(solver='liblinear')

# 2. Define the hyperparameter grid
param_grid = {
    'penalty': ['l1', 'l2'],
    'C': [0.01, 0.1, 1, 10, 100]
}

# 3. Set up GridSearchCV
grid_search = GridSearchCV(
    estimator=model,
    param_grid=param_grid,
    cv=5,  // 5-fold cross-validation
    scoring='accuracy',
    n_jobs=-1  // Use all available CPU cores
)

# 4. Fit to the training data
grid_search.fit(X_train, y_train)

# 5. Inspect results
print(f"Best parameters found: {grid_search.best_params_}")
print(f"Best cross-validation accuracy: {grid_search.best_score_:.3f}")

RandomizedSearchCV

Performs a random search, which is more efficient for larger hyperparameter spaces.

from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import uniform

# Define hyperparameter distributions
param_dist = {
    'penalty': ['l1', 'l2'],
    'C': uniform(loc=0, scale=100) // Sample from a continuous range
}

random_search = RandomizedSearchCV(
    estimator=model,
    param_distributions=param_dist,
    n_iter=50, // Try 50 random combinations
    cv=5,
    scoring='accuracy',
    n_jobs=-1
)
random_search.fit(X_train, y_train)

Choosing a Search Strategy

  • Grid Search: Best for small, discrete search spaces. Guarantees finding the best combination within the grid but can be very slow.
  • Random Search: More efficient for larger search spaces and often finds a better model in less time. The preferred choice in most cases.
  • Bayesian Optimization: An intelligent search method that is often the most efficient for computationally expensive models (like deep neural networks). Libraries like `scikit-optimize` provide this.
